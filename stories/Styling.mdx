<Title>Styling</Title>

`JBTimeInput` is a composed component. The time-input host owns the popover shell and renders a `JBButton` close action, while the inner `JBInput`, `JBPopover`, `JBTimePicker`, and `JBButton` keep their own reusable styling APIs.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Time-input recipes should use those common tokens where possible, then compose the existing `JBInput`, `JBPopover`, `JBTimePicker`, and `JBButton` APIs instead of recreating child component internals.

## Styling Model

Use these layers when styling time-input examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-time-input/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-time-input/stories/styles).
- Input recipes are reused from [`modules/jb-input/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-input/stories/styles).
- Popover recipes are reused from [`modules/jb-popover/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-popover/stories/styles).
- Time-picker recipes are reused from [`modules/jb-time-picker/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-time-picker/stories/styles).
- The close action uses `JBButton`; customize its exported `close-button` part with `--jb-button-*` variables documented in [JBButton Styling](../../jb-button/stories/Styling.mdx).
- Public time-input variables use `--jb-time-input-*` on the host. Use [JBInput Styling](../../jb-input/stories/Styling.mdx), [JBPopover Styling](../../jb-popover/stories/Styling.mdx), [JBTimePicker Styling](../../jb-time-picker/stories/Styling.mdx), and [JBButton Styling](../../jb-button/stories/Styling.mdx) as the source of truth for child component variables.

Each example uses one shared theme pack, the existing `JBInput` and `JBTimePicker` recipes, and a time-input recipe that maps the matching `JBPopover` shell values and `JBButton` close-action variables onto exported parts.

## Recommended Customization Path

1. Use `--jb-time-input-*` variables for host margin and popover stacking.
2. Reuse an existing `JBInput` recipe for the text-field layer. See [JBInput Styling](../../jb-input/stories/Styling.mdx) for the full input API.
3. Reuse an existing `JBPopover` recipe for the overlay shell. See [JBPopover Styling](../../jb-popover/stories/Styling.mdx) for background, radius, padding, shadow, and overlay behavior.
4. Reuse an existing `JBTimePicker` recipe for the wheel picker content. See [JBTimePicker Styling](../../jb-time-picker/stories/Styling.mdx) for the full picker API.
5. Style the close action through `::part(close-button)` with `--jb-button-*` variables. See [JBButton Styling](../../jb-button/stories/Styling.mdx) for its ghost-variant, size, border, and state variables.
6. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
7. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
8. Add a new time-input variable only when a needed host surface cannot be customized with current public variables or official parts.

```css
  --jb-button-border-radius: 999rem;

  box-shadow: none;
}
```

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles). Composed components should reuse child component recipes instead of copying their variables into the parent recipe.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-input/stories/styles/style-carbon.css */
.theme-name-input {
  /* reusable jb-input recipe */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-popover/stories/styles/style-carbon.css */
jb-popover.theme-name-popover {
  /* reusable jb-popover recipe */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-time-picker/stories/styles/style-carbon.css */
.theme-name-time-picker {
  /* reusable jb-time-picker recipe */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-time-input/stories/styles/style-carbon.css */
.theme-name-time-input {
  /* time-input host and jb-popover shell variables */
}

.theme-name-time-input::part(close-button) {
  /* jb-button variables for the close action */
}
```
